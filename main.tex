% Abstract

\makeabstracttitle

Computing systems have evolved dramatically, yet the security foundations of mainstream operating systems (OSes) remain rooted in protection models conceived decades ago.
Traditional OSes enforce security through process- and file-based abstractions, mediating access between subjects and objects under a monolithic kernel trusted computing base.
While this legacy model has proven durable, it no longer aligns with modern software complexity and emerging hardware protection mechanisms.
Contemporary processors introduce primitives such as sub-process isolation (e.g., Intel PKU, ARM MTE, CHERI) and trusted execution environments (TEEs, Confidential VMs), which redefine the boundaries of trust and isolation.
However, these advances often operate independently of OS enforcement, creating misalignments and new attack surfaces.

This thesis argues that sustainable system security requires systematically integrating hardware-backed protection into existing OS architectures.
It explores two key challenges: (1) principled retrofitting of hardware-based security into legacy OS abstractions, and (2) redefining OS responsibilities under hardware-enforced isolation.
To this end, the thesis presents \capacity, which retrofits capability-based security into in-process domains using hardware features such as pointer authentication and memory tagging; and \incognitos, which reimagines the OS as a self-obfuscating entity within Confidential VMs to mitigate side-channel leakage.
These systems illustrate how principled retrofitting enables better security within and beyond traditional process boundaries.

This thesis contributes to bridging the widening gap between legacy operating system architectures and the evolving landscape of hardware-enforced security.
By demonstrating how hardware protection mechanisms can be systematically retrofitted into established OS abstractions, it offers a practical pathway for strengthening system security without disrupting decades of software compatibility.


These approaches influence both research and practice: they provide a blueprint for extending the lifespan and trustworthiness of existing OSes, inform the design of future security architectures that align with modern hardware, and advance the broader goal of achieving provable security guarantees in commodity systems.


Its ideas contribute to the long-term vision of secure computing infrastructures where hardware and software collaborate seamlessly to defend against contemporary and emerging threats.




\textbf{Keywords:} Security, Operating System, Protection, Side channels
